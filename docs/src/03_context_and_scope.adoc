== Context and Scope

Esta sección describe el contexto del sistema YOVI, es decir, su entorno,
los actores externos con los que interactúa y las interfaces que delimitan
el alcance del sistema.

A diferencia de la sección de *Introduction and Goals*, aquí no se describen
requisitos ni objetivos de calidad, sino las fronteras del sistema y
su relación con el entorno.

=== Business Context

YOVI es una plataforma web que permite jugar partidas básicas del juego de
tablero Y contra un bot. El sistema actúa como intermediario entre los
jugadores humanos, el motor de juego automático y el almacenamiento de
resultados de las partidas.

Desde el punto de vista del negocio, el sistema se considera una *caja negra*
que ofrece una experiencia de juego sencilla y automatizada.

Los principales actores externos que interactúan con el sistema son:

[options="header",cols="1,2,2"]
|===
| Actor / Sistema externo | Inputs hacia YOVI | Outputs desde YOVI

| Jugador
| Nombre de usuario, movimientos de la partida
| Confirmaciones, estado del tablero, resultado de la partida

| Motor GameY
| Estado del tablero en notación YEN
| Movimiento calculado del bot

| Base de Datos
| Resultado final de la partida
| Confirmación de persistencia
|===

El jugador interactúa únicamente con la aplicación web.
El motor GameY no tiene conocimiento del usuario ni de la interfaz gráfica;
su responsabilidad se limita al cálculo del siguiente movimiento.
La base de datos almacena únicamente el **resultado final de cada partida**,
sin gestionar estadísticas avanzadas ni rankings en esta versión.

=== Technical Context

Desde el punto de vista técnico, el sistema está compuesto por tres
subsistemas independientes que se comunican mediante interfaces REST
y mensajes JSON.

[options="header",cols="1,2,2"]
|===
| Comunicación | Canal / Protocolo | Formato

| Webapp ↔ Users Service
| HTTP REST (puerto 3000)
| JSON

| Webapp ↔ GameY
| HTTP REST (puerto 4000)
| JSON (notación YEN)

| Users Service ↔ Base de Datos
| Conexión directa (driver de BD)
| Datos estructurados
|===

La aplicación web se ejecuta en el navegador del usuario y constituye el
punto de entrada único al sistema.
Los servicios backend no exponen interfaces gráficas y están diseñados
para ser consumidos únicamente mediante API.

=== Scope (Alcance)

El alcance de esta primera entrega (versión 0.1) está limitado a una
**prueba de concepto funcional**, cuyo objetivo es validar la arquitectura
y la integración entre los distintos subsistemas.

**Dentro del alcance:**

* Registro básico de usuarios para identificar las partidas.
* Juego de una partida básica del juego Y contra un bot.
* Comunicación funcional entre Webapp, Users Service y GameY.
* Almacenamiento en base de datos del **resultado final de la partida**.
* Aplicación desplegada y accesible vía web.

**Fuera del alcance (versiones futuras):**

* Rankings y estadísticas agregadas de usuarios.
* Almacenamiento del histórico completo de movimientos.
* Múltiples estrategias o niveles de dificultad del bot.
* Partidas multijugador entre personas.

Este alcance permite demostrar la viabilidad técnica del sistema y sirve
como base para futuras extensiones funcionales.
